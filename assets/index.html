What is Apache Nifi?

  • Next

Apache Nifi is a powerful, open-source data flow management system that automates the flow of data between software systems. It provides a user-friendly, web-based interface that allows users to easily create, control, and monitor data flows through a drag-and-drop interface.

How to configure Nifi?

These tools and components work together to create powerful and flexible data flows within NiFi. Users can drag and drop components onto the canvas, connect them to define the data flow logic, and configure each component's properties to meet their specific requirements. NiFi's intuitive user interface and modular design make it accessible to users with varying levels of technical expertise. The ability to create custom processors and leverage the NiFi expression language further enhances its versatility, allowing users to tailor the platform to their unique data processing needs.

Nifi's user interface is indeed designed to be intuitive and user-friendly, it's important to recognize that NiFi is a powerful and feature-rich data integration platform. The simplicity of the UI should not be mistaken for a lack of complexity or capabilities under the hood.

Processors are the building blocks of NiFi flows. They perform specific tasks on the data, such as extracting, transforming, routing, or persisting it. Input/Output Ports act as entry and exit points for data within a NiFi flow. Process groups are containers that logically group related processors and other components together. They help organize and manage complex flows by encapsulating subsets of the overall data processing logic. Remote process groups enable communication and data transfer between different NiFi instances or clusters. Funnels are used to combine or split data streams within a flow. They can merge data from multiple sources into a single stream or distribute data from a single source to multiple destinations. The Import from Registry feature enables users to import pre-built flow templates or components from the Registry into their NiFi instance. Labels are visual aids that help users annotate and document their NiFi flows.

Apache Nifi Home Page

Apache Nifi home page looks pretty minimal and the UI is very user friendly with the ease of use.

Nifi Home

The default landing page of Apache NiFi is https://localhost:8443/nifi, which serves as the entry point for accessing the NiFi services and running your projects. Upon navigating to this URL, you'll be prompted with a login page. The default username and password can be found in the logs/nifi-app.log file. Once logged in, the NiFi user interface presents a canvas-like workspace where you can design and orchestrate your data flows. The interface is divided into several panels:Components Panel: Located in the top-left corner, this panel provides a collection of available processors, controllers, and other components that you can drag and drop onto the canvas to build your data flow. Navigate Panel: Situated on the left side, this panel displays a hierarchical view of your data flow, allowing you to navigate through different process groups and components easily. Operate Panel: This panel, typically located on the right side, offers various controls to start, stop, pause, or terminate your processors. It also provides options to delete processors or perform other operational tasks. To begin your project, you can start by adding the InvokeHTTP processor from the Components Panel.

Step 1: Add InvokeHTTP processor

The first step in the NiFi flow is to configure the InvokeHTTP processor, which acts as the entry point for ingesting Bitcoin price data. Double-clicking on the processor allows you to specify the HTTP URL of the Bitcoin price API in the "Remote URL" field. In this project, we are ingesting historical and real-time Bitcoin price data, including attributes like date, opening price, highest price, lowest price, and closing price. The API is configured to return 10 entries per second, resulting in approximately 600 requests per minute, ensuring a steady data flow without overwhelming the API. It is recommended to provide a descriptive label, such as "Fetch Bitcoin Price Data," to clearly indicate the processor's purpose within the NiFi flow.

Step 2: Adding the JoltTransformjson Processor

The retrieved data from the Bitcoin price API was in JSON format, and the SplitJson processor was employed to transform and restructure this JSON data into the desired format. This processor can be configured to perform intricate transformations on the JSON data structure using the EvaluateJsonPath processor and a Jolt specification, a powerful JSON-to-JSON transformation language. By specifying the Jolt specification code in the processor's settings, we can extract desired fields, reshape, filter, and transform the JSON data according to our requirements. Apache NiFi's visual interface, modular design, and processors like SplitJson and EvaluateJsonPath empower us to handle complex JSON data integration tasks with ease and efficiency.

Step 3: Adding PutFile processor

The PutFile processor in Apache NiFi offers a versatile solution for persisting transformed data to various destinations. In this project, it was configured to write the extracted Bitcoin price data, transformed into a structured JSON format, to a local directory. However, NiFi's flexibility extends beyond local storage, enabling seamless integration with cloud platforms and remote destinations. The PutFile processor can be easily reconfigured to store data on cloud storage services like Amazon S3, Azure Blob Storage, or Google Cloud Storage, ensuring data accessibility and scalability. Furthermore, NiFi supports a wide range of output formats, including CSV, which was leveraged in this project to convert the JSON data into a CSV format suitable for analysis and visualization tools like Tableau. This versatility empowers users to tailor the data persistence process to their specific requirements, streamlining the integration of data into various downstream systems and applications.

Using Tableau to analyze the generated data

The NiFi flow culminated in the transformation of the JSON data into a CSV format, which was subsequently ingested into Tableau for comprehensive data analysis and visualization. The resulting visualization depicted the high, low, and opening prices of Bitcoin, with the x-axis representing the price range and the y-axis illustrating the monthly fluctuations in Bitcoin's value. This visual representation facilitated a deeper understanding of the historical price trends, enabling further exploration and insights into the cryptocurrency market's dynamics.

More on Nifi

Apache NiFi is renowned for its user-friendly interface and minimal coding requirements, making it accessible to users with varying technical expertise. When connecting processors within a data flow, it is crucial to define the relationships between them, such as success, failure, termination, response, and retry. In our case, the first processor is linked to the second one, with the success and retry checkboxes selected. This configuration instructs the first processor to pass the fetched data to the subsequent processor for further processing upon successful execution, while also enabling it to retry the operation in case of any updates or changes to the data source. NiFi's visual approach and intuitive relationship management streamline the process of building robust and scalable data pipelines, enabling seamless integration and automation across diverse systems and environments.
